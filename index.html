Sesion 2: Introduciendonos en HTML

¿Que es html 5?

Es la nueva versión del lenguaje de marcado que se usa para estructurar páginas web, que actualmente todavia sigue en su evolucion, gracias a él con características nuevas y modificaciones que mejorará significativamente este nuevo estándar.

¿Qué para que sirve?

El html5 no cambia este objetivo, solo que tendremos más oportunidad de hacer las cosas más simplificadas y de una manera más sencilla. Eso es todo, ahora podemos crear aplicaciones web de una manera menos complicada ya que esta tecnología mejora e incorpora muchos conceptos de programación web, como decir un todo en uno.

¿Porqué es mejor HTML5?

Simplemente porque es una tecnología que supera a la actual HTML, porque es lo nuevo que estandariza la W3C, porque es nueva tecnología y como toda nueva tecnología siempre viene con cosas que nos van a impresiona

¿Porqué debo de comenzar a trabajar con HTML5?

La respuesta es muy corta: debes de comenzar a crear tus proyectos en HTML5 porque es una inversión en el futuro, ¡¡ NO que digo futuro!! es para ¡¡YA..!!., la mayoria de los sitios web poderosos del mundo ya implementan este estandar, y si no quieres quedarte atras, debes de comenzar ya.

¿Qué hay de nuevo entonces en html5?

Hay muchas mejoras y cosas nuevas que nos pueden sorprender por ejemplo el tema de formularios, la forma de estructuración en fin… A continuación alguna de las reglas establecidas para HTM:

  • Las nuevas características debe basarse en HTML, CSS, DOM y JavaScript.
  • Reducir la necesidad de plugins externos (como Flash).
  • Mejor manejo de errores.
  • Más marcado para reemplazar secuencias de comandos.
  • HTML5 debe ser independiente del dispositivo.
  • El proceso de desarrollo debe ser visible para el público

Etiquetas

Cuando creamos sitios webs utilizamos etiquetas HTML para los contenidos. Estás etiquetas le dan al contenido un significado y le permite al navegador el poderle mostrar a los usuarios la estructura apropiada del sitio.

Para la creación de un sitio web es necesario conocer los dos tipos de estructura al que pertenecen:

  • Estructura de etiquetas: Los elementos que utilizamos para para describir el contenido
  • Estructura Semántica: Es la que provee información adicional.

Etiquetas de texto

Encabezados

Los navegadores despliegan los contenidos de encabezados en diferentes tamaños. Para el caso de las etiquetas de HTML, la etiqueta <h1></h1>, es por defecto, el más grande mientras que los contenidos que pertenecen a las etiquetas <h6></h6>, son los más pequeños.

Los tamaños exactos en los que el navegador despliega los encabezado varía según que navegador el usuario esté utilizando. Sin embargo, se puede crear tamaños globales utilizando CSS, que ya veremos más adelante.

Parrafos

Para crear un párrafo sólo es necesario envolver las palabras que son parte del párrafo en etiquetas <p></p>

Negrita e Itálica

Al envolver palabras con las etiquetas <b></b> podemos hacer que los caracteres se vean en negrita. Este elemento también representa una sección de la estructura que puede ser distinta visualmente. Sin embargo, esta etiqueta no da ningún tipo de significado semántico para el navegador. Otra opción mucho más semántica es la etiqueta Strong

Citas

Existen dos etiquetas que son las más usadas para las citas

<blockquote>:

Esta etiqueta es utilizada para aquellas citas que son largas como un párrafo. Por defecto, los navegadores tienden a indentar el contenido envuelto en esta etiqueta. Sin embargo, no es recomendable utilizar este elemento sólo para crear este efecto visual.

Cambios en el contenido

ETIQUETAS : LISTAS

HTML nos proporciona tres diferentes tipos de listas:.

  • Listas Ordenadas: aquellas listas donde cada item en la lista está ordenado de forma numérica. Por ejemplo, podríamos utilizar una lista de ingredientes para la receta de un postre.
  • Listas Desordenadas: aquellas listas que comienzan con viñetas, en vez de caracteres que muestran algún orden.
  • Listas Definidas: aquellas listas que tienen términos para cada item

Listas Ordenadas

Estas listas son creadas con la etiqueta <ol></ol>

Listas Desordenadas

Estas listas son creadas con la etiqueta <ul></ul>

Nota: Tanto para las listas ordenadas y desordenadas, se utiliza la etiqueta <li></li> para definir los items de la lista. Por defecto, los navegadores le ponen sangría al contenido de las listas

Listas de Definición

Las listas de definción se crean con la etiqueta <dl> y por lo general se compone de una serie de términos y definiciones. Dentro de las etiquetas <dl> usualmente se encontrarán pares de etiquetas <dt> y <dd>

<dt>: Esta etiqueta se utiliza para contener el termino definido.

<dd>: Esta etiqueta se utiliza para contener la definición.

Listas anidadas

Es posible colocar una lista dentro de una etiqueta <li> para crear una sublista o lista anidada.

Los navegadores desplegan las listas anidadas de forma indentada, alejadas un poco de la lista padre así como se cambia el estilo de las viñetas.

Enlaces

Los enlaces ( links ) son una característica importante de la web ya que nos permiten mover de un sitio web a otro, permitiendo hacer real el concepto de navegación. Usualmente nos encontraremos con los siguientes tipos de enlaces

  • Enlaces de un sitio web a otro
  • Enlaces de una página a otro en el mismo sitio web
  • Enlaces de alguna parte de una página web a otra parte de la misma página
  • Enlaces que se abren en una nueva página del navegador

¿Cómo se escriben los enalces?

Los enlaces son creados con la etiqueta <a> Los usuarios pueden hacer click en cualquier parte del texto que se encuentra dentro de estas etiquetas. Podemos especificar hacia que página queremos enlazar usando el atributo href

Por defecto, los navegadores despliegan los links con un color azul y subrayados.

<a href="http://www.google.com">Google</a>

Enlazando a otros sitios

Cuando queremos enlazar hacia otro sitio web, el valor del atributo href tiene que ser la dirección completa del sitio a donde queremos ir, más conocida como una URL o ruta absolutas

URLs Absolutas

El acrónimo URL significa Localizador Uniforme de Recursos ( Uniform Resource Locator ). Todo sitio web tiene su propio URL y es la ruta que uno escribe en el navegador para llegar a dicho sitio.

Las URLs absolutas comienzan con el nombre del dominio del sitio y seguido de la ruta a una página específica. En caso de no haber una página especificada el sitio mostrará la página de inicio.

Agregando Imágenes

Para empezar a agregar una imagen lo primero que necesitamos es la etiqueta <img>. Esta etiqueta se conoce como un elemento vacío, quiere decir que no tiene una etiqueta que lo cierra. Sin embargo, esta etiqueta debe de contener los siguientes atributos

src: Este atributo le dice al navegador donde puede encontrar la imagen. El valor de este atributo es usualmente una URL relativa que apunta hacia la imagen.

[mostrar video o gif con ejemplo de imagen]

alt: Este atributo nos permite dar una descripción de la imagen. Esta descripción será visible en caso de que la imagen no pueda cargar. El texto que se encuentra como valor de este atributo debe de ser lo más preciso posible para que así los lectores de pantalla y los motores de búsqueda, usados por personas con discapacidad visual, puedan entenderlo.

title: Este atributo puede ser usado como otra fuente de información de la imagen.

height y width: En algunos casos podemos apreciar estos atributos que determinan el alto y el ancho de la imagen respectivamente.

Tablas

Una nos sirve para mostrar información en forma de una grila en donde cada bloque se conoce como una celda. Las tablas nos ayudan a desplegar información tabulada. Por ejemplo, podemos mostrar reportes financieros, programación de televisión, etc.

La estructura básica de una tabla consiste en las siguientes etiquetas

  • <table>: Etiqueta que crea la tabla
  • <tr>: Etiqueta que crea las filas de la tabla
  • <td>: Etiqueta que representa la celda de una tabla. Usualmente va dentro de la etiqueta <tr>
  • <th>: Etiqueta que cumple la misma función de celda que <td> pero además sirve para que sea como celda de encabezado

Formularios

Tradicionalmente, el término "formulario" se ha referido a un documento que contiene espacios para que el usuario rellene información.

HTML toma este concepto para referirse al conjunto de etiquetas que nos permiten recolectar información de los usuarios que visitan nuestro sitio web.

Así necesitemos colocar una caja de búsqueda o crear multiples formularios para una aplicación, los formularios en HTML nos dan una serie de etiquetas que nos permiten recolectar data de los usuarios. En este unidad aprenderemos:

  • Como crear un formulario para nuestro sitio web
  • Diversas herramientas para recolectar data
  • Formularios HTML5

Controles de Formularios

  • Agregando Texto

    • Entradas de texto: Utilizado para ingresar una línea de texto como una dirección de correo eletrónico o nombres.

    Entradas de contraseña: Utilizado para ingresar una línea de texto pero enmascara los caracteres ingresados.

    Areas de Texto: Para áreas largas de texto como mensajes o comentarios.
  • Tomando Decisiones

    • Botones Radiales: Cuando el usuario necesita seleccionar una sóla opción.

    Casillas de Verificación: Cuando el usuario puede seleccionar y deseleccionar una o más opciones.

    Casillas desplegables:

  • Presentar Formularios

    • Botones de Enviar: Para enviar data desde el formulario hacia otro sitio web.

    Botones de Imagen: Con una función similar al de los botones de enviar pero con la diferencia de que se pueden enviar imagenes.

    Subiendo archivos: Para subir archivos directamente a un sitio web.

Estructura de Formularios

Para comenzar a estructurar un formulario necesitamos la etiqueta <form>. Esta etiqueta soporta diferentes atributos, todos de manera opcional:

Nombre del Atributo Descripción
action Todo formulario requiere de este atributo. Su valor es la URL de la página del servidor que recibirá la información una vez que el formulario sea enviado
method

Al momento de enviar un formulario podemos utilizar dos métodos: get o post.

method

Con este método, los valores del formulario son agregados a la URL que se especifica como valor en el atributo action. Este método es ideal para los siguientes casos:

  • Formularios pequeños ( ejemplo: buscadores )
  • Cuando estás recibiendo data desde el servidor web ( no enviando información que puede ser agregada o eliminada desde la base de datos )

post

Con este método, los valores son enviado como cabeceras HTTP. Como regla, debes de usar este método si tu formulario:

  • Le permite a los usuarios subir archivos
  • Es muy largo
  • Contiene data sensible ( contraseñas, números de tarjeta de crédito, etc )
  • Agrega y/o elimina información desde una base de datos.

Nota: En caso de no haber un método definido en el formulario, la información será enviada, por defecto, con el método get
id

Así como usamos este atributo como identificador de otras etiquetas, también podemos usarlo para identificar de manera única a un formulario de los demás. Además, el valor es utilizado con scripts.

Para conocer sobre más atributos de la etiqueta puedes ingresar a la Red de Desarrolladores de Mozilla

Entradas de Texto

Etiqueta <input>

Esta etiqueta es utilizada para crear diferentes categoria de controles para el formularios. El valor del atributo type determina el tipo de entrada se va a crear

type="text"

Cuando el valor del atributo type es text crea una entrada simple de una sola línea.

name

Cuando los usuarios ingresan información en un formulario, el servidor necesita saber de que control cada dato es ingresado. Por ejemplo, en un formulario de login el servidor necesita saber el dato que se ha ingresado como usuario y como contraseña. Debido a esto, cada control de formulario necesita un atributo name. El valor de este attributo sirve como identificador.

maxlength

Podemos usar este atributo para limitar el número de caracteres que el usuario vaya a ingresar.

name

El valor del atributo indica el nombre de la entrada de contraseña. De esta manera, el servidor recibe la contraseña que el usuario va a ingresar y puede hacer una mejor validación.

Areas de Texto

Etiqueta <textarea>

Esta etiqueta es usada para crear entradas multilineales. A diferencia de la etiqueta input esta etiqueta no es vacía. Es decir, tiene un elemento de apertura y de cierre.

Todo texto que aprezca entre esta etiqueta aparecerá en la entrada una vez que la página se cargue en el navegador.

Botones Radiales

type="radio"

Los botones radiales le permiten a los usuarios escoger una sóla opción

name

Este atributo es enviado al servidor con el valor de la opción que el usuario selecciona. Cuando una pregunta se le hace al usuario con la opción de escoger diversas respuestas con botones radiales el valor de este atributo debe de ser el mismo para todos los botones radiales de todas las respuestas.

value

Este atributo indica el valor que es enviado al servidor para la opción seleccionada. El valor de cada uno de los botones en el grupo debe de ser diferente para que el servidor sepa cual de ellos el usuario ha seleccionado.

checked

Este atributo sirve para indicar si un valor debería de estar seleccionado por defecto cuando la página cargue. Su valor es checked

Lista desplegable de cajas

Una lista desplegable de cajas, también conocida como una caja seleccionable, nos permite seleccionar una opción desde una lista desplegable utilizando la etiqueta <select> </select> que a su vez contiene las opciones envueltas en la etiqueta <option> </option>

name

Este atributo indica el nombre del control de formulario que será enviado al servidor junto con el valor que el usuario seleccionó

value

La etiqueta <option> </option> contiene este atributo para indicar el valor que es enviado al servidor

selected

Este atributo puede ser usado para indicar la opción que debería estar seleccionada por defecto una vez que la página se carga. Su valor es selected

Si este atributo no es utilizado por defecto la primera opción será mostrada una vez que la página cargue

La función de estas listas desplegables es similar a los botones radiales. Podemos usarla para el caso donde tenemos una lista muy larga de opciones.
